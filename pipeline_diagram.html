Pipeline Components at a Glance

Training
Initial Training Pipeline
Extract → Feature Engineering → SMOTE balancing → Train 3 models (RF, XGBoost, LR) → Validate → Promote best to MLflow production alias. Optuna hyperparameter tuning available via do_tuning=True.
Serving
FastAPI Inference Server
Model loaded at startup via FastAPI lifespan. Background task polls MLflow every 5 min and reloads silently when a new model is promoted — no user request ever waits for a download.
Monitoring
Three-Layer Drift Monitoring
Layer 1 — Feature drift (DataDriftPreset)
Layer 2 — Prediction drift (ColumnDriftMetric)
Layer 3 — Performance drift via ClassificationPreset (self-activates on first customer feedback row)
Retraining
Tiered Retraining Response
🔴 Serious ≥50% features OR prediction drift OR recall <70% → retrain immediately

🟡 Negligible 15–49% features → Sunday midnight schedule

🟢 None <15% drifted → no action
Alerts
Slack Notifications
Team is notified on every daily monitoring run regardless of outcome. Messages include drift score, prediction drift status, model performance metrics, and whether retraining was triggered.
🔄
Feedback
Option B — Performance Monitoring
Every prediction is logged to a predictions table. When chargebacks/disputes populate fraud_feedback, ClassificationPreset activates automatically — no code changes needed.
Storage
Persistent Storage Layer
MinIO — models, preprocessor, reference baseline
MLflow — experiments, model registry, aliases
PostgreSQL — transactions, predictions, fraud feedback
Infrastructure
Docker Compose Stack
PostgreSQL · MinIO · MLflow Server · Prefect Server + Worker · FastAPI · Prometheus · Grafana. All services health-checked with proper dependency ordering.
